Plugin Architecture

What you're building today

.claude/
├── .claude-plugin/
│   └── manifest.json    # Plugin metadata
├── skills/              # Reusable prompts
│   └── my-skill/
│       └── SKILL.md
├── hooks/               # Event-triggered actions
│   └── hooks.json
├── agents/              # Specialized sub-agents
│   └── researcher.md
└── commands/            # Slash commands
    └── review.md

Skills

Reusable workflows with structured output

Hooks

Trigger on events: pre-commit, post-tool, session-start

Agents

Specialized sub-agents for focused tasks

Hooks: Event-Driven Automation

Run code when things happen

pre-commit

Validate before committing

Lint, test, security checks

post-tool-use

React after Claude uses a tool

Log, notify, validate output

session-start

Initialize when Claude starts

Load context, check state

notification

Handle notifications

Slack alerts, email summaries

// hooks/hooks.json
{ "hooks": { "pre-commit": [{ "command": "bash scripts/lint.sh" }] } }

Agents: Specialized Sub-Agents

Focused expertise for complex tasks

Researcher Agent

Deep dives into codebases, docs, tickets

Returns structured findings

Code Review Agent

Focused PR review with checklists

Security, performance, style

Task Orchestrator

Breaks down complex work

Parallelizes, tracks progress

Agents have their own system prompts, models, and tool access

Getting Started

One command to scaffold everything

cd workshop/my-plugin-day-02
./init.sh

# or
make setup

This will:

  • Check dependencies (gh, claude)
  • Clone the plugin scaffold
  • Ask for your plugin name
  • Start Claude Code

Then talk to Claude:

"Build my plugin"

"Create a skill for X"

"Add a pre-commit hook"

"Validate my plugin"
